数据流与 API 设计

数据如何从 Shopify 加载、编辑、保存

📥 加载流程

// 使用 Shopify Admin GraphQL API 一次加载所有产品及其 SEO 数据
query LoadProducts($first: Int!) {
  products(first: $first, sortKey: TITLE) {
    nodes {
      id
      title
      handle
      seo {
        title
        description
      }
      featuredImage { url }
    }
    pageInfo { hasNextPage endCursor }
  }
}
说明：首次加载最多 250 个产品，如果超过 250 个则进行分页。 SEO 数据直接从 seo { title, description } 字段获取。 表格编辑都在前端内存中完成，不触发额外 API 请求。

💾 保存流程

// 批量保存 — 一次性提交所有修改
mutation BulkUpdateSEO($input: ProductUpdateInput!) {
  productUpdate(product: $input) {
    product { id seo { title description } }
    userErrors { field message }
  }
}

// 所有修改逐条发送，但由前端统一控制进度
// 显示 "正在保存 (3/5)..." 的进度条
说明：点击"保存全部修改"后，逐条调用 productUpdate， 前端显示保存进度（如 3/5 完成）。成功/失败的结果汇总后显示 toast 通知。

⚠️ 错误处理

🟡 字段超长
实时显示红色警告 + 字符计数，但允许保存（Shopify 本身也会截断）
🔴 保存失败
逐条报告哪些产品保存成功/失败，失败的可单独重试
🟢 Handle 冲突
修改 Handle 时检测重复，保存前校验，实时提示
